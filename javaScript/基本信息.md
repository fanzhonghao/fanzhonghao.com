#### 关于JavaScript
JavaScript和Java没有任何关系，仅是为了沾Java的光才这样命名。其中还仿照了一些Java的结构语法。
JavaScript是浏览器的脚本语言，Java是编写应用程序的高级语言。JavaScript和其他语言一样，都应用于高级任务控制，大多是解释执行，都属于弱类型语言，数据类型在运行时决定，实现自动内存管理机制，资源的分配策略相似。但不同的是，JavaScript运行于浏览器，主要用于Web开发。
#### 脚本语言
脚本语言是一种应用程序扩展语言，用于系统的扩展，使其按用户的意愿去运行。所有的基础功能由系统提供，脚本语言在更高层次描述如何调用系统的接口。和其他编程语言不同，脚本语言通常不需要经历编译和链接这些阶段，都多可以直接执行。也有的语言需要编译，但这是为了执行得更快一点。

### 对象篇
	JavaScript不是一种面向对象的语言，因为它没有提供面向对象所具有的一些明显特征，如继承和多态。
    JavaScript设计者将它称为“基于对象”而不是”面向对象“的语言。
    JavaScript中没有类的概念，创建一个对象只要定义一个该对象的构造函数并通过它创建对象即可。
    使用函数对象的this指针可以为函数对象动态添加属性。
    JavaScript是动态语言，没有早绑定即静态绑定。JavaScript的对象的类型在运行时才能确定。

#### 框架操作
HTML中的frameset元素可以创建框架。虽然在很多时候都把框架称为Frame对象，但是事实上，在JavaScript中不存在Frame对象。所谓的Frame对象只是window对象的一个实例，该对象拥有windowwindow对象的所有方法和属性及事件。  

每个窗口都有一个name属性，其表示一个窗口名，可以通过窗口来索引一个窗口的引用。在多框架页中name属性可以取得框架窗口的引用，进行将一个新文档在指定窗口打开，如存在一个窗口集合如下：

    	<frameset>
        <frame src = "menu.html" name = "menu">//菜单窗口
        <frame src = "main.html" name = "main">//主窗口
        </frameset>

下面是个超链接，该链接属于menu.html文档，其带有一个target属性，指明将在哪个窗口中打开链接指向的文档，如下代码所示：

    <a href="Item1.html" target="main">
    当用户在menu窗口中单击该链接时，文档item1.html将在main框架窗口中打开

### 脚本化cookie
一个登录注册的用户在浏览某一网站时，需要在多个页面之间进行切换，用户的信息需要保存，否则每访问一个新页时都要重新登录。为了避免这一烦琐的过程，开发商在浏览器端使用了cookie技术，将用户信息临时保存起来。  
即cookie就是一些用户数据信息，只是它们以文件的方式保存起来，可以读取和修改。可以利用它与某个网站进行联系，并在浏览器与服务器之间传递消息。也即cookie的最经典的用途是保存状态，识别身份。  
每个cookie所存放的数据不会超过4kb，而每个cookie	文件中不会多余300个cookie。  
cookie的主要作用是保存状态，识别身份。因此在很多情况下都可以使用到cookie，特别是在用户身份验证的时候。例如，实现严格的跨页面全局变量。它的一般用途包括保存用户登录状态、跟踪用户行为、定制页面、创建购物车等。  
cookie虽然有以上优点，但是它也有缺点，如cookie可能被禁用，cookie是与浏览器相关的，不同的浏览器之间所保存的cookie也是不能相互访问的，cookie可能被删除及cookie安全性不够高等，所以使用时需要谨慎。  
在JavaScript中，创建cookie是通过设置cookie的键和值的方式来完成的。一个网站的cookie一般是不唯一的，可以有多个，而且这些不同的cookie还可以拥有不同的值。例如要存放用户名和密码。则可以用两个cookie，一个用于存放用户名，另一个用于存放密码。然后再使用document对象的cookie属性可以用来设置和读取cookie。每个cookie都是一个键/值对，如下所示：
> document.cookie="id=8";

如果要一次存储多个键/值对，可以用分号加空格（；）隔开，如下所示：
> document.cookie="id=12;us=yx";

获取cookie的值可以由document.cookie直接获得，示例如下：
> var strCookie = document.cookie;

这样，就可以获得以分号隔开的多个cookie键/值字符串。不过这样取得的键/值是指该域名下的所有cookie。  
cookie都是使用未编码的格式存入cookie文件中。但是在cookie中是不允许包含空格、分号、逗号等特殊符号的。如果要将这些特殊符号也写入cookie中，就必须在写入cookie之前，现将cookie用escape编码，再在读取cookie时通过unescape函数将其还原。  
在默认情况下，cookie是临时存在的。在一个浏览器窗口打开时，可以设置cookie，只要该浏览器窗口没有关闭，cookie就一直有效，而一旦浏览器窗口关闭后，cookie也就随之消失。  
如果想要cookie在浏览器窗口关闭之后还能继续使用，就需要为cookie设置一个生存期。所谓生存期就是cookie的终止日期，在这个终止日期到达之前，浏览器可以随时读取该cookie。一旦终止日期到达之后，该cookie将会从cookie文件中删除。  
cookie虽然是由一个网页所创建，但并不只是创建cookie的网页才能读取该cookie。在默认情况下，与创建cookie的网页在同一目录或子目录下的所有网页都可以读取该cookie。但如果在这个目录下还有子目录，要使在子目录中也可以访问，则需要使用path参数设置cookie，语法如下：
> document.cookie="name=value;path=cookieDir";

如果要使cookie在整个网站下可用，可以将cookieDir指定为根目录，示例如下：
> document.cookie="userId=320;path=/";

path不能解决在不同域中访问cookie的问题。在默认情况下，只有和设置cookie的网页在同一个Web服务器的网页才能访问该网页创建的cookie。但可以通过domain参数来实现对其的控制，语法格式如下：
> document.cookie="name=value;domain=cookieDomain";

在默认情况下，cookie都是采用不加密的HTTP的传输方法，这种方法容易被别人窃听。如果cookie中的信息很重要，就不能用这种方法了。因此，在JavaScript提供了cookie的secure属性，可以解决这个问题。  
当设置cookie的secure属性后，cookie就只能通过HTTP或其他安全协议来传输，这样消息就不容易被窃听。secure属性是一个布尔类型的值。  
设置了secure属性，只保证cookie与Web服务器之间的数据传输过程加密，而保存在本地的cookie文件并不加密。别人仍然可以从本机上查看cookie信息。  
使用cookie的注意事项：
* 由于cookie是存放在客户端的文件，可以使用第三方工具来查看cookie的内容。因此cookie并不是很安全。
* 每个cookie存放的数据最多不能超过4kb。
* 每个cookie文件最多只能存储300个cookie。
* cookie可能被禁用。
* cookie是与浏览器相关的。这意味着即使访问的是同一个页面，不同浏览器之间所保存的cookie也是不能互相访问的。
* cookie可能被删除。因为每个cookie都是硬盘上的一个文件，因此很有可能被用户无意间删除。
* cookie安全性不够高。所有的cookie都是以纯文本的形式记录于文件中，因此如果要保存用户名密码等信息时，最好事先经过加密处理。

### JavaScript与XML技术

#### XML简介
XML是eXtensible Markup Language的缩写，它是一种类似HTML的标记语言，用来描述数据的层次结构及存储数据。XML的数据标记不在XML中预定义，用户必须定义与数据相关的有意义的标记。XML语言需要专门的解释程序，通过分析XML提取数据。  
XML是一种描述数据结构的语言，与之相应的是XML语言解释器。如果没有解析器它所描述的数据就无法理解，同时也失去了意义。解释器提供的接口对程序员来说统称为API，最先出现针对XML的API是SAX（Simple API for XML），它是一套程序包。  
SAX提供了一套基于事件的XML解析的API。SAX解析器从XML文件的开头出发，每当遇到节点标签、文本或其他的XML语法时，就会激活一个事件。事件处理程序由应用开发人员编写，因此可以在事件处理程序中决定如何处理XML文件当前节点的数据。  

#### DOM
W3C的DOM规范制定了一系列标准用于描述结构化、层次化的数据，如HTML和XML。使用DOM接口处理XML文件是当前Web客户端开发常用的方法，大多数浏览器都实现W3C制定的DOM接口。  
DOM以树的形式组织文档中的数据，树的结构也就是ＨＴＭＬ或ＸＭＬ文档的元素节点为层次。便利一个文档中所有节点就是遍历DOM树的操作，第一个节点使用一个node对象来表示，该对象提供了操作节点的接口。document是最顶层的节点，所有的其他节点都是附属于它的。  
DOM模型是以XML为核心，所有遵循DOM规范的文档都可以使用DOM接口来处理。但HTML并没有完全遵循DOM规范。  
DOM的节点对象拥有从node对象继承而来的属性，也可以拥有自己独有的属性。这些属性可以用来存储一些与节点相关的数据，读取一个属性通常调用节点元素的getAttribute方法，设置某个属性的值通常调用节点元素的setAttribute方法。  
DOM接口提供了快速方便的方法访问一个指定的节点，如通过指定节点标签名、节点名称或节点ID来获得目标节点的引用。即：
* getElementsByTagName：该方法返回一个与指定标签名吻合的节点对象的引用，如果传入的标签名为“*”，则返回文档中所有的节点元素。
* getElementByName：该方法返回与指定name属性相吻合的元素集合。
* getElementById：该方法返回与指定ID相同的元素节点。

document对象创建新节点的方法如下：
* createTextNode，创建文本节点
* createDocumentFragment，创建文档碎片
* createElement，通过指定标签名创建节点

创建好节点后调用节点对象node的appendChild方法将新节点添加为某一个节点的子节点。  
JavaScript是动态语言，属性、方法、事件在其中称为特性。属性用于描述对象的状态，但在使用上却和事件一样，通过给事件赋予一个函数的地址即可完成绑定任务，操作方式和给属性赋值一样。在DOM中，通常使用getAttribute和setAttribute操作的特性，但特定于HTML的DOM为能更方便的操作对象的特性，将操作方式统一起来。  
节点迭代器NodeIterator，是遍历DOM树的辅助工具，用它可以对DOM树进行深度优先的搜索。如果要查找页面中某个特定类型的信息或元素，此迭代器非常有用。  
NodeIterator在使用前必须先创建它的对象实例，使用document对象的createNodeIterator()方法即可。  
另一个用于便利文档树的工具是TreeWalker，该对象拥有NodeIterator所有功能。另外TreeWalker还拥有一些NodeIterator没有的特性，它主要用于遍历文档树，访问树上的每一个节点。  

### 正则表达式
#### 正则表达式基础
正则表达式是用来检测字符串之间的匹配关系的工具，用这个工具可以轻松地确定两个字符串之间的匹配情况，可以用于二进制数据的匹配。  
RegExp是JavaScript提供的一个对象，用来完成有关正则表达式的操作和功能，每一条正则表达式模式对应一个RegExp实例。JavaScript使用RegExp对象封装与正则表达式相关的功能和操作，每一个该对象的实例对应一跳正则表达式。创建一个RegExp实例语法如下：
> var regObj = new RegExp("pattern" [,"flags"]);

参数说明：
* pattern：必选项，正则表达式的字符串
* flags：可选项，是一些标志组合
> 在标志组合中，"g"表示全局标志。设定时将搜索整个字符串，以找到匹配的内容，每一次新的探索都从RegExp对象的lastIndex标记的字符起，否则只搜索到第一个匹配的内容。"i"表示忽略大小写标志。

另一种创建方式为：
> var regObj = /pattern/[flags];

参数的意义与上一种一样，但是不能将pattern和flags括起来。使用方式如下：
> regObj.test(string);

#### 量词
量词是指定某个特定模式出现的次数。分为简单量词、贪婪量词、惰性量词和支配了量词。  
贪婪量词，它是首先匹配整个字符串，如果不匹配，则去掉最后一个字符，然后再比较。如果仍然不匹配，再继续去掉最后一个字符再比较，一直下去，直到找到匹配或字符串的字符被取完为止。支配量词，只尝试整个字符串的匹配，即只比较一次。  

#### 使用前瞻
前瞻是指对所要匹配的字符做一些限定条件。比如在检查用户输入的是否为电子邮箱地址时，其中有一个特殊的符号@，这就算是一个限定，即可以用前瞻的方法来确定用户输入的地址是否合法。前瞻分为正向前瞻和负向前瞻，正向前瞻是指在目标字符串的对应位置处要有指定的某一特殊的值。不过这个值不作为匹配结果处理，也不会存储在缓冲区内。负向前瞻与正向相反，是指在指定的位置不能有指定的值，它的处理结果也不作为匹配结果处理，也不会存储在缓冲区中。  

### Ajax基础
Ajax可以实现网页局部刷新，从而节省了大量的网络带宽并且提高了网络传输的速度。Ajax含义可以理解为异步JavaScript和XML，其核心是通过将少量数据用XML语言来描述并且在浏览器和服务器间进行异步传输。基于这一技术，网页可以实现动态刷新。  
传统的Web与服务器采用同步交互技术，用户表单完整地发送到服务器段进行处理后再返回一个新页面到浏览器。这一过程中因为网速延迟而导致浏览器有一定等待时间，交互体验比较差。为解决这一问题，Web开发者尝试使用异步通信组与服务器通信以改善这一问题，并取得了成功。这一套方案是使用XMLHttpRequest组件与服务器异步通信，在客户端用JavaScript实现网页内容的更新显示，后称为Ajax技术。  
Ajax的基本原理是在Web浏览器端使用JavaScript程序与服务器进行通信，并传递数据。JavaScript相对用户来说是完成数据的验证、响应用户的请求并与服务器异步传输数据。JavaScript使用XMLHttpRequest与服务器通信，该组件实现与服务器间的数据交换，数据结构通常是用XML语言描述。一个Ajax典型应用工作流程如下：
* JavaScript程序创建一个XMLHttpRequest对象
* 使用XMLHttpRequest对象向服务器发送HTTP请求
* 服务器响应请求并给客户端返回信息
* 客户端JavaScript程序在收到回馈信息后使用DOM等接口向用户呈现

Ajax的有点：
1. 更新数据时不需要刷新整个页面，只更新需要改变的数据，其他大量的不变的数据不需要传输，整体加快了互联网的运行速度
2. 每一次用户向服务器提交操作数据后，马上执行，浏览器不会等待一个更适合提交的时机。而Ajax可以先将数据保留在本地，等待满足一些条件后再向服务器发送或丢弃。并且可以先给用户UI一个快速的反馈。
3. 在一个类似不能注册相同用户的情况下，用户所填写的注册资料必须发往服务器后才知道所填写的用户名是否唯一。而Ajax可以事先利用空闲时间后台加载各种辅助数据，或者在用户填写好用户名后就在后台将其发往服务器进行检查。
4. Ajax是异步更新的，在Ajax引擎更新数据的同时，页面还响应用户的其他操作。

Ajax不是新开发出来的技术，而是已有的多种技术的再组合。  

Ajax的客户端技术，主要实现浏览器与服务器间的异步通信。通过基础通信组件将客户端数据发往服务器处理，并且接收从服务器返回的数据。客户端脚本程序从基础通信组件中获取数据并用于更新本地网页，通常只更新发生改变的部分，称为局部刷新。客户端脚本语言主要包括JavaScript、VBScript等，这些语言都可以用来实现Ajax的客户端功能。  
### JavaScript与插件
在编程语言的分类中，JavaScript属于自动化语言。与此类似的有VBScript、Visual Basic等，语言本身仅提供算术运算、逻辑运算和流程控制等功能。而真正复杂的与低层相关的工作则由语言之外的组件提供，因此自动化语言必须和其所在的环境相结合才能真正发挥作用。   
#### ActiveX控件
ActiveX是微软公司提出的一套二进制组件发布方案、实现规范和工具集合的总称。目的是实现软件二进制级别的兼容和复用。COM规范推出以后，Windows平台上只需遵循COM规范的软件都能相互访问并且实现二进制组件的复用。  
ActiveX控件本质上是一个COM服务器，自动化语言运行时库就是自动化客户机。  
ActiveX控件可以由任何语言实现，最后生成的组件只要遵循ActiveX技术规范即可。

### JavaScript的调试与优化
准备工作：
1. 打开IE浏览器，选择菜单栏“工具”|“Internet选项”命令，打开“Internet选项”对话框
2. 单机“高级”选项卡，拖动“设置列表框的垂直滚动条。在“设置”列表框中找到两个“禁用脚本调试”复选框，将它们前面的勾去掉。单机“确定”按钮。

进行调试：
1. 在IE中打开包含JavaScript程序的HTML网页，选择菜单栏“工具”|“开发人员工具”命令，打开“开发人员工具”对话框
2. 在工具栏中选择“脚本”调试，单机“启动调试”按钮进入调试

一般都在JavaScript程序中添加“debugger”语句来激活程序的调试，当浏览器执行到debugger语句时便启动调试。

设置断点：
1. 在代码编辑窗口中，将光标移动到需要添加断点的行上
2. 按下一次F9键，当前行的背景色变为红色，并且在窗口左边界上标上红色的圆点。
> 当程序在断点处暂停时，按下F5、F10或F11可以继续向下执行，F10可以逐过程执行，F11可以逐语句执行。
3. 将鼠标移动到源代码编辑窗口中的变量名上时，鼠标光标处将显示变量当前时刻的值。单击变量信息框中的变量值可以修改变量的当前值。

JavaScript使用自动内存管理机制。解释器回收内存的原则是，当内存不够用或机器空闲时运行内存管理程序。当对象引用链断开，也就是已经不再使用时就回收其所占的内存空间。因此，开发者应该在对象不再被使用时给引用对象的变量赋予“null”值，表明对象已经不在使用。
